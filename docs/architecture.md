# Архитектура системы

## Обзор

Маркетплейс построен по принципу микросервисной архитектуры:

- **python-backend** (port 8000) — основной API. Управляет пользователями, каталогом услуг и жизненным циклом заказов. При платёжных операциях делает синхронные HTTP-вызовы в go-payment через общий `httpx.AsyncClient` (создаётся в lifespan).

- **go-payment** (port 8001) — изолированный платёжный микросервис. Хранит состояние эскроу-счетов и ведёт блокчейн-лог транзакций. Данные персистируются в PostgreSQL (таблицы `blockchain`, `escrow_accounts`, `escrow_balances`) и восстанавливаются при перезапуске.

- **PostgreSQL 16** — единая СУБД. Схема `public` используется Python-бэкендом (asyncpg + SQLAlchemy). Go-сервис использует ту же БД через pgx/v5.

- **frontend** (port 3000) — статический HTML + ES6 на nginx:alpine. Взаимодействует с обоими сервисами напрямую из браузера.

## Блокчейн-лог

Каждая операция (lock/release/refund) порождает запись `Transaction`:

```
hash = SHA256(id + timestamp + from + to + amount + prev_hash)
```

Первая транзакция: `prev_hash = "0000000000000000"`. Каждая последующая содержит хэш предыдущей — это делает любое изменение исторической записи обнаруживаемым. Публичный эндпоинт `GET /chain/verify` пересчитывает все хэши и проверяет цепочку linkage.

Временны́е метки усекаются до микросекунды (`time.Microsecond`) для согласованного хэширования при round-trip через PostgreSQL (`TIMESTAMPTZ` хранит микросекунды).

## Принятые архитектурные решения

| Решение | Причина |
|---------|---------|
| Два сервиса вместо монолита | Изоляция платёжной логики, независимое масштабирование |
| PostgreSQL persistence в Go | Блокчейн и эскроу-состояние восстанавливаются после перезапуска |
| asyncpg + SQLAlchemy async | Неблокирующий I/O для FastAPI |
| Атомарное обновление баланса | `UPDATE ... SET balance = balance + :delta` — исключает race condition при конкурентных запросах |
| JWT Bearer (без cookies) | Совместимость с мобильными клиентами и API-потребителями |
| bcrypt для паролей | Индустриальный стандарт, устойчив к rainbow-table атакам |
| Rate limiting (in-memory) | 20 req/60s на IP для `/register` и `/login`; сбрасывается в тестах |
| Shared httpx.AsyncClient | Создаётся один раз в lifespan, переиспользуется для всех запросов к go-payment |
